# 项目自我进化方案

本文说明如何让 stock-agent 在运行中**根据结果自动调整行为**，实现一定程度的「自我进化」。项目已有推荐回测、记忆、RAG 等基础，可在此基础上增加闭环。

---

## 一、已有基础（可复用）

| 机制 | 位置 | 作用 |
|------|------|------|
| **推荐记录 + 回测** | `data/recommendations.py` | 每次报告 9/10 分买入写入 JSONL，并计算至今/1周/1月/2月/3月收益、胜率、基准对比、跌破卖出条数 |
| **回测展示** | `report/build_html.py` + 报告页 | 报告内展示「既往推荐表现」：胜率、收益分布、基准同期 |
| **长期记忆** | `chains/memory_store.py` | 深度分析结果存 JSONL，下次同标的分析时可「与上次对比」 |
| **RAG** | `rag/` | 报告卡片可写入向量库，综合分析时按 ticker 检索拼进 prompt |
| **可配置阈值** | `config/analysis_config.py` | `RECOMMEND_MIN_SCORE`、震荡市 `RECOMMEND_SIDEWAYS_MIN_SCORE`、技术指标等均可环境变量覆盖 |

---

## 二、进化方向与落地方式

### 1. 数据闭环：回测结果反哺评分（已实现一档）

**思路**：近期推荐胜率偏低时，本周期自动在评分 prompt 里加一句「请更保守给 9/10 分」，减少后续低质量推荐。

**实现**：

- 报告生成**前**先拉取「既往推荐表现」`get_past_recommendations_with_returns(since_days=90)`，得到 `recent_win_rate_pct`。
- 若 `recent_win_rate_pct < EVOLVE_WIN_RATE_THRESHOLD`（默认 45），在 `_build_prompt` 的【评分与动作要求】后追加：
  - `【策略反馈】近期推荐胜率偏低（X%），本周期请更保守：仅对基本面+技术面+消息面均非常突出的标的给 9 分，10 分保留给极少数。`
- 这样**无需改配置、无需人工**，下一份报告会自动收紧评分标准。

**配置**（可选）：

- `EVOLVE_WIN_RATE_THRESHOLD`：低于该胜率（%）时触发收紧，默认 45。
- `EVOLVE_ENABLED=0` 可关闭此行为。

---

### 2. 参数自适应（可扩展）

**思路**：用回测 summary 自动调节「记录推荐」的门槛。

- 当近期胜率持续 &lt; 40%：本周期临时把 `RECOMMEND_MIN_SCORE` 提高到 10（只记录 10 分买入），或提高 `RECOMMEND_SIDEWAYS_MIN_SCORE`。
- 实现方式：在 `server.py` 里根据 `backtest_summary` 计算 `min_score_override`（已有震荡市逻辑），再扩展为「近期胜率过低时也提高 min_score_override」。

---

### 3. Prompt 注入：报告总览与教训进 RAG

**思路**：每期报告总览（LLM 生成的 3～5 句摘要）和「本期教训」写入 RAG；下次生成报告前检索「上几期总览与教训」拼进 system 或 user prompt。

- 需要：报告生成后把 `report_summary` 与可选「教训」段落写入向量库（新文档类型）。
- 综合分析时：在现有 RAG 检索基础上，增加对「历史报告总览」的检索，拼成一段「过去几期要点与教训」供 LLM 参考。

---

### 4. A/B 测试评分标准（进阶）

**思路**：维护两套评分描述（如「保守版」与「标准版」），按周期或随机选一套；用回测胜率/收益统计哪套更优，长期倾向更优的那套。

- 需要：在配置或代码里区分 prompt 版本，记录「本报告使用的版本」到推荐记录或单独日志；分析时按版本聚合胜率。

---

### 5. 工程层面的自动维护（可选）

- **依赖与测试**：定时跑 `pip list --outdated`、`pytest`，失败时通知或阻断部署。
- **配置版本化**：对 `analysis_config`、prompt 模板做版本或 hash，便于回溯「某次表现变差是改了哪条配置」。

---

## 三、当前已实现的「自我进化」行为

- **回测胜率偏低时自动收紧评分**  
  - 条件：`EVOLVE_ENABLED` 非 0，且既往推荐「最近 N 条胜率」&lt; `EVOLVE_WIN_RATE_THRESHOLD`（默认 45%）。  
  - 行为：在本轮报告的**每只标的**的综合分析 prompt 中追加【策略反馈】一句，要求本周期更保守给 9/10 分。  
  - 配置：`config/analysis_config.py` 中 `EVOLVE_ENABLED`、`EVOLVE_WIN_RATE_THRESHOLD`，可通过环境变量覆盖。

---

## 四、小结

| 方向 | 状态 | 说明 |
|------|------|------|
| 回测胜率 → 收紧评分 prompt | ✅ 已实现 | 胜率低于阈值时自动在 prompt 中加「更保守」约束 |
| 回测 → 临时提高 RECOMMEND_MIN_SCORE | 可扩展 | 在现有 min_score_override 上按胜率再提一档 |
| 报告总览/教训进 RAG | 待实现 | 需要新文档类型与检索拼接逻辑 |
| A/B 评分标准 | 待实现 | 需要版本标识与按版本统计 |
| 依赖/测试自动化 | 待实现 | 与业务进化正交，属工程实践 |

通过「回测 → prompt 注入」这一档，项目已经具备**根据历史表现自动收紧评分**的自我进化能力；其余方向可按需逐步加。
